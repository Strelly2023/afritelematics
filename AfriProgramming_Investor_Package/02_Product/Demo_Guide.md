# Demo Guide

Open:

`afriprogramming_demo/index.html`

## Flow

1. Dashboard: controlled-pilot system overview.
2. Drift: contract violation detected.
3. Design: UML-derived state intent.
4. Replay: correct behavior proven through deterministic replay.
5. Contracts: candidate constraints shown.
6. Proposal: governance-ready, activation blocked.
7. Governance: explicit approval boundary.
8. Activation: controlled gate.
9. Runtime: resolved controlled simulation.

## Key Message

The system detects, proves, proposes, governs, and resolves without uncontrolled mutation.

## Phrases To Use

- Controlled pilot environment.
- Replay-proven behavior.
- Governed change.
- No automatic mutation.

## Phrases To Avoid

- Production-ready.
- Fully autonomous.
- Self-fixing.
- Guaranteed real-world reliability.
