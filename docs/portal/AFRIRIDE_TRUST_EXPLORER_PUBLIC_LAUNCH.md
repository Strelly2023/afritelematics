# AfriRide Trust Explorer Public Launch

Status: TRUST EXPLORER PUBLIC LAUNCH
Classification: PUBLIC_TRUST_SURFACE_LAUNCH_PLAN

Purpose: define how to launch the AfriRide Trust Explorer publicly as a bounded
registry and verification visualization surface.

This launch plan is a public-facing rollout surface.
It is not proof of unrestricted global public adoption.

## Launch Goal

Expose a public-facing Trust Explorer that lets partners, investors, and
operators inspect:

- registry entries
- verification visualization
- first partner narratives
- protocol and packaging cues

without turning the UI into runtime authority.

## Public Launch Rule

```text
the Trust Explorer may visualize registry and verification evidence
the Trust Explorer may not certify truth independently of replay
```

## Launch Sequence

### Phase 1. Curated Evidence Launch

- publish curated registry entries from bounded evidence
- publish verification visualization panels
- publish authority boundary note

### Phase 2. Partner Demo Launch

- add first-partner use cases
- add onboarding pathway
- add commercial packaging summary

### Phase 3. Public Protocol Positioning

- link protocol spec
- link developer portal
- link partner onboarding surfaces

## Required Public Panels

- Trust Explorer Registry
- Verification Visualization
- First 5 Partners
- Monetization Surface
- Protocol Components

## Required Public Copy

- replay-linked
- registry indexes evidence
- bounded external verification
- not a second truth layer

## Launch Success Metrics

- Trust Explorer demo sessions booked
- portal visits from partner targets
- registry packet walkthrough completion rate
- first public protocol inquiry

## Authority Boundary

This launch plan permits only this bounded claim:

```text
AfriRide can launch the Trust Explorer publicly as a replay-linked registry and
verification surface
```
