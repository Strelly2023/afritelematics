STATUS: INVESTOR MODEL FRAMEWORK

## Valuation Principle

AfriRide valuation should rise with evidence quality.

```text
Architecture creates defensibility.
Pilot evidence creates credibility.
Transactions create valuation power.
```

## Stage-Based Valuation Logic

| Stage | Description | Indicative Range |
| --- | --- | --- |
| Architecture only | Strong platform thesis, scaffold-ready modules, no field proof | USD 0.5M-2M |
| Working MVP + pilot | Installable apps, live backend, replay-admissible field evidence | USD 2M-7M |
| Live traction + payments | Real rides, repeat users, real payments, earnings ledger | USD 7M-20M+ |
| Multi-city traction | Multiple active cities, operational revenue, partner proof | USD 25M+ |

## Evidence Multipliers

Positive valuation drivers:

- replay-admissible field execution
- repeat rider and driver usage
- real payment volume
- fleet/business customers
- signed city or transport partnerships
- defensible compliance/reporting workflow

Valuation discounts:

- no field evidence
- no live transactions
- no repeat usage
- mocked payment providers only
- regulatory claims without approvals
- architecture not wired into production runtime
