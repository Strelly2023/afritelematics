# Protocol Lifecycle

1. Execution request (`fn_id`)
2. Gossip propagation
3. Deterministic execution
4. Proof generation
5. Consensus agreement
6. Trust evaluation
7. Block commit
8. Chain verification
9. State projection
10. Service exposure
11. Observability export

## Activation Boundary

Protocol readiness does not imply live pilot authorization. Live authorization requires passing external backend, device, operator, replay, and evidence gates.
